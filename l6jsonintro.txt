=> JSON ( Javascript Object Notation )

=JSON
Javascript Object Notation
Base on Text Format
Light-weighted
Does not support comments(commandပေးတာကိုပြောတာ) and namespace
NoSQL database like MogoDB use JSON formats


=>JSON
Need to enclose name of key with double quotes" (should not use single quotes ')
But can use single quotes ' to enclose key names

=>JSON vs XML
JSON is more consise/lightweight/easy to understand
XML is more text longer and look more complex


=JSON Syntax Rules 
JSON data to written inside curly braces 
Data is represented as key valude pairs
Keys should enclose in double quotes "
Key and value should be separated by colon(:)
Comma(,) is used to separated data
